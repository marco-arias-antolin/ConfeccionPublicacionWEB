Identificación de las partes que intervienen en la verificación del script

Enunciado

Durante el proceso de desarrollo de código, ya sea una página web, un micro servicio, etc., e independientemente del lenguaje empleado, es importante saber cómo analizar el código, detectar los errores y solucionarlos.

Para practicarlo, le solicitamos:

  • Analice el siguiente código detenidamente.
  • Corrija los errores que pueda detectar.
  • Describa las partes que intervienen en el mismo.
<html>

<head>
	<meta charset=”utf-8”>
	<title>Ejercicio de grupo tema 3</title>
</head>

<body>
	<script>
		// esta versión tiene errores de sintaxis
		var input = “”
		var input = window.prompt(escribe un contenido,”*”);
		document.write =”input”;
	</script>
</body>

</html>

Descripción de las partes del código:

  • <html>: Define el comienzo del documento HTML.
  • <head>: Contiene la información del encabezado del documento, como la codificación de caracteres y el título de la página.
  • <meta charset="utf-8">: Especifica la codificación de caracteres del documento como UTF-8, que es ampliamente utilizado y compatible con varios idiomas.
  • <title>: Define el título de la página que se mostrará en la pestaña del navegador.
  • <body>: Contiene el contenido visible del documento HTML.
  • <script>: Permite incluir código JavaScript en el documento HTML.
  • var input = "";: Declara una variable llamada input y le asigna una cadena vacía como valor inicial.
  • var input = window.prompt("Escribe un contenido", "*");: Abre un cuadro de diálogo donde el usuario puede ingresar un contenido, y luego asigna ese contenido a la variable input. El asterisco (*) es el valor predeterminado que se muestra en el cuadro de diálogo.
  • document.write(input);: Escribe el valor de la variable input en el documento HTML.

Actividad asociada al CE1.1

Fases de verificación de la integración de componentes en páginas web:

  1. Diseño: En esta fase se define la estructura y el diseño de la página web, incluyendo la identificación de los componentes necesarios.
  2. Desarrollo: En esta fase se crea el código fuente de la página web, incluyendo los componentes individuales y su lógica.
  3. Pruebas unitarias: Se realizan pruebas en cada componente de forma individual para verificar su funcionamiento correcto.
  4. Integración: Los componentes individuales se combinan y se integran en la página web completa.
  5. Pruebas de integración: Se realizan pruebas para verificar que los componentes integrados funcionen correctamente en conjunto y cumplan con los requisitos.
  6. Depuración: Si se encuentran errores durante las pruebas de integración, se identifican y corrigen para asegurar un funcionamiento adecuado.
  7. Despliegue: Una vez que la página web ha pasado todas las pruebas y los errores han sido corregidos, se implementa y despliega en un entorno de producción o en el servidor correspondiente.
  8. Mantenimiento: Se realizan actualizaciones y correcciones adicionales según sea necesario durante la vida útil de la página web para garantizar su correcto funcionamiento.
