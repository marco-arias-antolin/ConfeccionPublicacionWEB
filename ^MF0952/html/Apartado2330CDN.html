Identificación de funciones y características de las herramientas de transferencia de archivos

Enunciado

Para cargar o descargar archivos en un servidor FTP es necesario un FTP client, cliente FTP en castellano. Este tipo de programa construye una conexión TCP al puerto del servidor, donde, por medio de comandos, se realiza el intercambio de datos. Muchos navegadores de internet cuentan con un FTP client incorporado, sin embargo, también existen programas FTP especiales para Windows, macOS y otros sistemas operativos que simplifican significativamente la transferencia de datos con el protocolo FTP. Uno de ellos es Filezilla.

A continuación, debe establecer una conexión FTP con el servidor utilizando FileZilla. Después, deberá transferir la carpeta Almacenamiento Principal, con todos sus subdirectorios, que se creó en ejercicios anteriores.

Transferencia de un archivo

Actividad asociada al CE2.2

El comando FTP (Protocolo de Transferencia de Archivos) en Linux es una herramienta de línea de comandos que se utiliza para realizar operaciones de transferencia de archivos entre un cliente y un servidor a través del protocolo FTP. Aquí tienes algunos de los comandos y órdenes más comunes que proporciona esta herramienta:

  1. ftp: Inicia una sesión FTP con un servidor remoto. 
    ftp hostname
  2. open: Abre una conexión a un servidor remoto. 
    open hostname
  3. user: Establece el nombre de usuario para autenticación en el servidor remoto. 
    user username [password]
  4. pass: Ingresa la contraseña del usuario en el servidor remoto. 
    pass password
  5. ls: Lista los archivos y directorios en el directorio remoto actual. 
    ls [remote-directory]
  6. cd: Cambia el directorio remoto actual. 
    cd remote-directory
  7. get: Descarga un archivo desde el servidor remoto al cliente local. 
    get remote-file [local-file]
  8. put: Sube un archivo desde el cliente local al servidor remoto. 
    put local-file [remote-file]
  9. mget: Descarga múltiples archivos desde el servidor remoto al cliente local. 
    mget remote-files
  10. mput: Sube múltiples archivos desde el cliente local al servidor remoto. 
    mput local-files
  11. delete: Elimina un archivo en el servidor remoto. 
    delete remote-file
  12. mkdir: Crea un nuevo directorio en el servidor remoto. 
    mkdir directory-name
  13. rmdir: Elimina un directorio en el servidor remoto. 
    rmdir directory-name
  14. rename: Cambia el nombre de un archivo o directorio en el servidor remoto. 
    rename old-name new-name
  15. quit: Cierra la sesión FTP y sale del cliente FTP. 
    quit

Estos son solo algunos de los comandos más comunes que puedes usar en el cliente FTP de Linux. Ten en cuenta que la disponibilidad y el comportamiento exacto de los comandos pueden variar según la implementación específica del servidor FTP al que te estás conectando. Puedes obtener más información sobre los comandos específicos y sus opciones ejecutando ftp y luego usando el comando help o ? dentro de la sesión FTP para obtener una lista completa de los comandos disponibles.
