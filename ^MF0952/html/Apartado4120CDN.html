Identificar los entornos de prueba para verificación

Enunciado

Una vez que ha subido una página web a un servidor es necesario:

  1. Comprobar que los archivos se han subido correctamente.
  2. Comprobar el funcionamiento de la web en el servidor.
  3. Detalle el proceso a seguir para cada una de estas acciones con capturas de pantalla.
  4. Identifique los entornos de prueba que va a utilizar para la verificación y funcionamiento de la página web.

    • Entorno local: Utilizar un servidor local en tu propia computadora.
    • Entorno de desarrollo: Plataformas como WordPress, Joomla o Drupal.
    • Entorno de pruebas o staging: Configurar un servidor de pruebas independiente.

    • Herramientas de seguridad y vulnerabilidad: Análisis de seguridad y pruebas de penetración.

Actividad asociada al CE3.1

Los siguientes entornos de prueba se utilizarán para verificar la página web de acuerdo con las especificaciones recibidas:

  • Entorno local: Para pruebas de diseño y funcionalidad en una máquina local.
  • Entorno de desarrollo: Para verificar cambios en el CMS y la aplicación web antes de su lanzamiento.
  • Entorno de pruebas o staging: Para realizar pruebas exhaustivas de nuevas características y actualizaciones antes de la implementación.
  • Emuladores y simuladores: Para verificar la compatibilidad en diferentes navegadores y dispositivos virtuales.
  • Navegadores de desarrolladores: Para inspeccionar y depurar código, y probar la respuesta en tiempo real.
  • Dispositivos físicos: Para verificar la apariencia y el rendimiento en dispositivos reales.
  • Herramientas de prueba de rendimiento: Para evaluar la velocidad de carga y el rendimiento general.
  • Herramientas de accesibilidad: Para garantizar la accesibilidad según las pautas WCAG.
  • Herramientas de seguridad y vulnerabilidad: Para identificar y abordar posibles problemas de seguridad.
  • Entorno de pruebas de usuarios reales: Para obtener retroalimentación de usuarios antes del lanzamiento oficial.
