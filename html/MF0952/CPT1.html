Utilización de comandos de creación para crear elementos en estructura de almacenamiento

Enunciado

Desde el terminal de Windows cree una carpeta llamada Almacenamiento Principal en el escritorio, dentro de la misma cree otras dos carpetas llamadas Partición 1 y Partición 2. Dentro de Partición 1 cree un archivo .txt donde deberá escribir: “Hola, soy un archivo de texto nuevo”.

  • Utilice los comandos adecuados para la creación de esta estructura de archivos.
  • Verifique los permisos de acceso a los directorios creados.
  • Modifique el directorio “partición2” para que esté contenido dentro de “partición 1”
  • Genere un nuevo directorio denominado “partición3” dentro de “partición 2”
  • Elimine el directorio “partición2”
  • Identifique los posibles errores que pueden surgir en la creación y modificación de directorios.
  • Documente con capturas de pantalla todo el proceso para la corrección para la corrección por parte de su tutor/a.

Actividad asociada al CE1.3:

En un caso práctico, en el que se dispone de un servidor web, con acceso a un directorio en el que contamos con permiso de creación, modificación y eliminación de elementos, para crear la infraestructura de almacenamiento del sitio web siguiendo unas especificaciones recibidas:

  • Verificar los permisos de acceso al directorio.
  • Crear los elementos de la estructura de almacenamiento según especificaciones recibidas.
  • Utilizar los comandos de creación, modificación y eliminación de elementos contenedores en la estructura de almacenamiento.
  • Identificar errores en la creación y modificación de elementos en el sistema de almacenamiento.
  • Documentar los procesos realizados.

Solución

Abre el terminal de Windows (presiona Win + R, escribe cmd y presiona Enter) y navega al escritorio con el siguiente comando:

cd %userprofile%\Desktop

Crea la carpeta "Almacenamiento Principal" y las subcarpetas "Partición 1" y "Partición 2" con los siguientes comandos:

mkdir "Almacenamiento Principal"
cd "Almacenamiento Principal"
mkdir "Partición 1"
mkdir "Partición 2"

Ingresa a la carpeta "Partición 1" y crea un archivo de texto con el contenido indicado:

cd "Partición 1"
echo Hola, soy un archivo de texto nuevo > archivo.txt

Verifica los permisos de acceso a los directorios:
Puedes utilizar el comando icacls para verificar los permisos.
Por ejemplo, para verificar los permisos de "Almacenamiento Principal":

icacls "%userprofile%\Desktop\Almacenamiento Principal"

Modifica la ubicación de "Partición 2" para que esté contenida dentro de "Partición 1". Primero, asegúrate de regresar al escritorio:

cd %userprofile%\Desktop

Luego, mueve la carpeta "Partición 2" dentro de "Partición 1":

move "Almacenamiento Principal\Partición 2" "Almacenamiento Principal\Partición 1\Partición 2"

Genera un nuevo directorio "Partición 3" dentro de "Partición 2":

mkdir "Almacenamiento Principal\Partición 1\Partición 2\Partición 3"

Elimina la carpeta "Partición 2":

rmdir /s /q "Almacenamiento Principal\Partición 1\Partición 2"

Posibles errores:

  • Asegúrate de tener los permisos necesarios para crear, modificar y eliminar archivos y carpetas en la ubicación que deseas.
  • Los nombres de archivo y carpeta deben estar escritos correctamente y no contener caracteres especiales que puedan causar problemas en el sistema de archivos de Windows.

Captura de Pantalla:

Actividad asociada al CE1.3:

Verificar los permisos de acceso al directorio:

Puedes usar el comando ls -l para verificar los permisos de acceso de los elementos en el directorio:

ls -l

Crear la estructura de almacenamiento según las especificaciones:

Supongamos que las especificaciones son crear una carpeta llamada "images" y dentro de ella otra carpeta llamada "photos", junto con un archivo HTML llamado "index.html":

mkdir images
mkdir images/photos
touch index.html

Modificar y eliminar elementos en la estructura:

Si deseas modificar el archivo "index.html", puedes usar un editor de texto en línea de comandos como nano o vim. Por ejemplo:

nano index.html

Para eliminar la carpeta "photos" y el archivo "index.html":

rm -r images/photos
rm index.html

Identificar errores:

  • Asegúrate de tener los permisos necesarios en el directorio /var/www/html para crear, modificar y eliminar archivos y carpetas.
  • Verifica que los nombres de archivos y carpetas sean correctos y no contengan caracteres especiales que puedan causar problemas en el sistema de archivos.
  • Al modificar o eliminar elementos, asegúrate de estar en el directorio correcto para evitar cambios no deseados.
